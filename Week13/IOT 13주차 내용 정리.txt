1. 센서와 실습 환경

	온습도/조도 센서가 달린 모듈에 펌웨어(프로그램)를 올려서
	무선 센싱 실습 진행

	펌웨어 올리기 전 환경 구축은 시그윈(Cygwin)으로 이미 완료
		→ 시그윈 환경에서 컴파일과 업로드 진행

	터미널에서 cd /opt/tinyos-2.x/apps/Blink 명령어로
	예제 디렉토리 확인 가능

2. 실습 파일 관리

	교수님 깃허브에서 교안과 예제 파일 다운

	다운 받은 파일은 Cygwin/home/pc 같은 위치에 복사

	이렇게 하면 시그윈 환경에서 바로 파일 사용 가능 (편리)

3. TinyOS와 nesC 구성

	파일 종류	역할
	AppC.nc	와이어링 파일 (컴포넌트 선언 및 연결)
	.h 파일	헤더 파일 (상수, 구조체 등 공유 데이터 정의)
	.nc 파일	모듈 구현 (실제 코드 로직)

4. 컴포넌트 예시

	SensirionSht11C() : 온습도 센서용 라이브러리

	IlluAdcC : 조도 센서 관련 라이브러리

	BatteryC : 배터리 상태 관리 라이브러리

5. enum과 #define 차이

	enum : 그룹으로 묶어서 상수 관리 가능, 컴파일러가 타입 체크해줌

	#define : 단순 상수 치환, 컴파일러 타입 체크 불가

6. 상수 예시 (Test.h)

	상수 이름	의미	값 (예시)
	TEST_PERIOD	센서 동작 주기	10240 = 약 10초 (1024 = 1초, 61440 = 1분)

7. Makefile

	컴파일, 업로드 시 설정하는 옵션들

	채널 번호: 0 ~ 26

	그룹 번호: 0 ~ 255

	필요에 따라 변경 가능

8. 컴파일 및 펌웨어 업로드

	make telosb : 컴파일 (에러 발생 시 표시됨)

	USB로 모듈 연결 후, 노드 아이디 확인 (우리는 모니터 화면 번호를 함)

	make telosb install <노드ID> : 펌웨어를 모듈에 업로드(퓨징)

	무선 Zigbee 네트워크로 센서 모듈 작동 시작
